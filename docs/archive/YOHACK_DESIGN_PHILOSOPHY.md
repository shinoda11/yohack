# YOHACK デザイン哲学 v1.0

## コアコンセプト：「決断の静けさ」

人生最大の買い物を前にした人は、煽られたくない。急かされたくない。
ただ、静かに正確に考えたい。YOHACKのUIはその「考える場所」である。

ペルソナは日常的にApple、Porsche、Aesopを選ぶ人たちだ。
細部の雑さを0.5秒で見抜く。愛されるプロダクトは細部に意志がある。

## 3つの設計原則

### 原則1: 削ることが設計である
表示できる情報を全て表示しない。
その画面で「一つだけ」伝えることを決める。
余白は空白ではなく、重要なものを際立たせるための設計。

実装指針:
- 1画面に主役は1つ。それ以外は脇役か非表示
- コンポーネントの padding は現状の1.5倍を基準にする
- 「あった方がいい」要素は削る。「なければ困る」要素だけ残す

### 原則2: 数字は人生の重さを持つ
8,500万円は単なる数字ではない。その人の20年分の決断だ。
金額・スコア・年齢の表示には、その重みに相応しいタイポグラフィと余白を与える。

実装指針:
- 最重要数値: text-4xl以上、DM Sans Bold、#5A5550
- 金額は万円・億円単位で表示、細かい端数は見せない
- 数値の変化には必ずtransitionを付ける（0.3s ease）
- 重要数値の周囲には必ず余白を確保（他の要素と詰めない）

### 原則3: 応答が信頼を作る
入力が変わればグラフが動く。
その応答の速さと滑らかさが「このツールは生きている」という信頼になる。
アニメーションは装飾ではなく、因果関係の表現。

実装指針:
- 入力変更→シミュレーション更新は視覚的に明示（ローディング状態）
- グラフの線は一瞬で描かれるのではなく、左から右へ流れる
- ボタンのhover/activeは必ず実装。反応のないUIは死んでいる
- エラーと空状態は必ず設計する。未定義の状態を作らない

## カラーの意味

| 色 | コード | 意味 | 使用場面 |
|---|---|---|---|
| ウォームゴールド | #C8B89A | 決断・信頼 | CTA、重要アクション、ロゴ |
| ダークテキスト | #5A5550 | 落ち着き・誠実 | 本文、見出し |
| オフホワイト | #FAF9F7 | 静けさ・余白 | 背景 |
| セーフグリーン | #4A7C59 | 安全・安心ライン以上 | 安全圏のグラフ、完了状態 |
| ウォーニングレッド | #CC3333 | 危険・要注意 | 安心ライン以下、エラー |

## タイポグラフィの意味

- DM Sans Bold: 数字・結論・見出し（重要なことを言う時）
- DM Sans Regular: UI要素・ラベル（機能を説明する時）
- Noto Sans JP: 日本語本文（寄り添って話す時）

## 参考プロダクト

- **craft.do**: 余白の使い方、フォントへのこだわり
- **Linear**: 情報密度が高いのに煩雑に見えない
- **Copilot Money**: 金融データを不安を煽らず正確に見せる
- 日本の建築事務所サイト: ノイズの排除、紙の質感、決断の重さ
